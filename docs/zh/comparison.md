## 详细对比

### 对比 aider

**aider** 是一个专注于 git 仓库内代码编辑的 AI 结对编程工具。

| 方面 | zhi | aider |
|------|-----|-------|
| 定位 | 文档处理 + 通用终端 AI | git 仓库中的代码编辑 |
| 文件编辑 | 写入隔离的 `zhi-output/` 目录 | 原地编辑文件，自动提交 |
| OCR | 内置（通过智谱 API 处理 PDF、图片） | 无 |
| 技能 | 15 个可复用 YAML 工作流 | 无对应功能 |
| LLM | 智谱 GLM（针对中文优化） | 多提供商（OpenAI、Anthropic 等） |
| Git 集成 | 通过 shell 工具 | 深度 git 集成 |

**选择 zhi**：文档处理、中文任务和可复用工作流。
**选择 aider**：带 git 集成的 AI 代码编辑。

### 对比 Claude Code

**Claude Code** 是 Anthropic 官方的 CLI 代理，由 Claude 模型驱动。

| 方面 | zhi | Claude Code |
|------|-----|-------------|
| 定位 | 文档处理 + 中文 LLM | 代码代理 + 通用开发 |
| LLM | 智谱 GLM | Anthropic Claude |
| 成本 | 双模型（技能使用 flash，约 10% 成本） | 单模型定价 |
| 技能 | 15 个 YAML + 可组合 | 斜杠命令（不可组合） |
| 文件安全 | 隔离输出目录 | 沙箱 + 审批提示 |
| 中文 | 原生 GLM 支持 | 以英文为主 |

**选择 zhi**：中文文档工作流和高性价比批量处理。
**选择 Claude Code**：利用 Claude 推理能力的代码开发任务。

| 方面 | zhi | GitHub Copilot CLI |

## zhi 的独特优势

### 原生中文 LLM + 文档处理

zhi 是唯一将原生中文 LLM（智谱 GLM）与完整文档处理流水线（包括 OCR、表格提取和格式转换）结合的终端 AI 工具。这使其成为中文办公工作流的理想选择。

### 双模型架构

zhi 策略性地使用两个模型：

- **GLM-5**（高级）用于交互对话 -- 智能、细致的回答
- **GLM-4-flash**（经济）用于技能执行 -- 快速、高性价比（约 10% 的成本）

这意味着批量操作（如总结 50 个文档）的成本仅为单模型方案的一小部分。

### YAML 技能系统

技能是可组合的 YAML 配置，而非硬编码功能。您可以：

- 无需编写代码即可创建自定义技能
- 组合技能（例如 `contract-review` 使用 `analyze` + `compare` + `proofread`）
- 以 `.yaml` 文件形式分享技能

### 输出隔离

与原地编辑文件的工具不同，zhi 将所有输出写入隔离的 `zhi-output/` 目录。您的原始文件永远不会被修改，可以放心尝试而不必担心风险。
